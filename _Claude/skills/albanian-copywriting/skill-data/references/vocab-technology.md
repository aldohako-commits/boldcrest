## IT & Technology Vocabulary

### Hardware Architecture & Systems (Hardueri & Arkitektura e Sistemeve)

| **Albanian Term** | **English Equivalent** | **Definition/Usage** |
|---|---|---|
| **Arkitektura e kompjuterit** | Computer architecture | The complete set of rules and methods that describe the functioning, organization, and implementation of computer systems; the Von Neumann architecture is the foundation model |
| **Arkitektura Von Neumann-it** | Von Neumann architecture | Named after mathematician John von Neumann (1940s); still used in most modern computers; consists of CPU, memory, input/output devices, and bus systems |
| **Njësia Qendrore e Përpunimit (CPU)** | Central Processing Unit | Electronic circuit responsible for executing instructions of a computer program; often called "the brain" of the computer; also known as **mikroprocesor** or **procesor** |
| **Procesor/Mikroprocesor** | Processor | Alternative terms for CPU; producers include Intel and AMD |
| **Cikli fetch-execute** | Fetch-Execute Cycle | The basic operation cycle of CPU: Fetch instruction → Decode instruction → Execute instruction |
| **Marrja e udhëzimit** | Instruction Fetch | First step of fetch-execute cycle; CPU retrieves instruction from main memory and stores in temporary register (CIR) |
| **Çkodifikimi i udhëzimit** | Instruction Decode | Second step; CPU must understand the instruction to execute it |
| **Zbatimi i udhëzimit** | Instruction Execution | Third step; CPU performs the operation specified by the decoded instruction |
| **Program Counter (PC)** | Program Counter | Register containing the address of the next instruction to be fetched |
| **Regjistri i Adresave të Kujtesës (MAR)** | Memory Address Register | Register that holds the memory address being accessed |
| **Regjistri i Të Dhënave të Kujtesës (MDR)** | Memory Data Register | Register that holds data transferred to/from memory |
| **Regjistri Aktual i Udhëzimeve (CIR)** | Current Instruction Register | Register that holds the instruction currently being executed |
| **Akumulatori (AC)** | Accumulator | General-purpose register for arithmetic and logic operations |
| **Njësia e Kontrollit (CU)** | Control Unit | Monitors information flow in system and executes program instructions; coordinates all components |
| **Njësia Aritmetike Logjike (ALU)** | Arithmetic Logic Unit | Performs arithmetic calculations, logical operations, data classification and searching |
| **Regjistrat** | Registers | Temporary storage locations in CPU; hold data and instructions being processed |
| **Bus-et / Sistemi Bus** | Bus/System Bus | Physical connections transmitting data between computer components; composed of: address bus, data bus, control bus |
| **Bus kontrolli** | Control Bus | Carries control signals between CPU and other devices |
| **Bus i të dhënave** | Data Bus | Carries actual data between components |
| **Bus i adresës** | Address Bus | Carries memory addresses |
| **Memoria kryesore** | Primary Memory / Main Memory | RAM; temporary storage for data and instructions currently in use; volatile (lost when power off) |
| **Memoria sekondare** | Secondary Memory | Long-term storage (hard drives, SSDs, flash drives); non-volatile; slower access than primary |
| **DRAM** | Dynamic RAM | Type of primary memory; cheaper, more common; requires constant refresh |
| **SRAM** | Static RAM | Type of primary memory; faster, more expensive; used for cache |
| **Memoria Cache** | Cache Memory | Very fast memory between CPU and main memory; improves performance; smaller capacity than main memory |
| **PROM** | Programmable Read-Only Memory | Non-erasable read-only memory |
| **EEPROM** | Electrically Erasable PROM | Can be erased and reprogrammed electronically |
| **Memoria Flash** | Flash Memory | Non-volatile memory; used in USB drives, SD cards; faster access than traditional magnetic storage |
| **SSD** | Solid State Drive | Storage device using flash memory; faster than mechanical hard drives; no moving parts |
| **Shpejtësia e orës** | Clock Speed | Speed at which CPU executes instructions; measured in GHz (gigahertz); 1 GHz = 1 billion cycles per second |
| **GHz (Gigahertz)** | Gigahertz | Unit of measurement for CPU clock speed; modern CPUs reach 4.7 GHz or higher |
| **Dual core** | Dual Core | Processor with two cores; can execute multiple tasks simultaneously |
| **Quad core** | Quad Core | Processor with four cores |
| **Multi-core** | Multi-Core | Processor with multiple processing cores; performs parallel processing (**përpunimi paralel** or **Pipeline**) |
| **Bërthamat** | Cores | Individual processing units within a multi-core processor |
| **Përpunimi paralel** | Parallel Processing / Pipeline | Multiple cores executing different tasks simultaneously; more powerful than single-core |
| **RISC** | Reduced Instruction Set Computer | CPU architecture with simplified instruction set; faster execution |
| **CISC** | Complex Instruction Set Computer | CPU architecture with complex instruction set |
| **Graphics Processing Unit (GPU)** | Graphics Processing Unit | Specialized processor for handling graphics and intensive calculations |
| **Performance** | Performance | How well a computer executes tasks; determined by CPU speed, cache, core count, and other factors |

### Operating Systems & System Software (Sistemet Operative & Softueri i Sistemit)

| **Albanian Term** | **English Equivalent** | **Definition/Usage** |
|---|---|---|
| **Sistem operativ (SO)** | Operating System | Software that controls overall functioning of computer; enables user interaction with hardware; executes applications; examples: Windows, macOS, Linux |
| **Funksionet e SO-së** | OS Functions | Control hardware; manage memory; manage processes; handle input/output; provide user interface; manage file systems |
| **Ndërfaqja e përdoruesit** | User Interface | System through which users interact with computer; can be GUI (graphical) or CLI (command-line) |
| **GUI (Graphical User Interface)** | Graphical User Interface | Visual interface using windows, icons, menus; user-friendly; most common today |
| **CLI (Command Line Interface)** | Command Line Interface | Text-based interface; user types commands; more powerful but less intuitive |
| **Programet e shërbimeve (Utility)** | Utility Programs | System software that performs specific maintenance or support tasks |
| **Data Recovery** | Data Recovery | Utility for recovering deleted or lost files |
| **Disk Cleanup** | Disk Cleanup | Utility for removing temporary and unnecessary files to free space |
| **Disk Defragmentation** | Disk Defragmentation | Utility for reorganizing fragmented data on disk for faster access |
| **Programet me burim të hapur** | Open Source Programs | Software with source code publicly available; users can modify and distribute (e.g., Linux, Python) |
| **Programet me kod të mbyllur** | Proprietary/Closed Source Programs | Software owned by company; source code not publicly available; users cannot modify |
| **Proprietary** | Proprietary | Commercial software with closed source; licensed to users |
| **Freeware** | Freeware | Free software; can be used without cost; source code may not be available |
| **Shareware** | Shareware | Software distributed for free trial; paid license usually required for full features |
| **Përkthyesit (Translators)** | Translators | Programs converting high-level language code to machine code |
| **Compiler / Përpilues** | Compiler | Translator that converts entire source code to machine code before execution; produces executable file |
| **Interpreter / Interpretues** | Interpreter | Translator that converts and executes code line-by-line; no executable file created |
| **Assembler / Bashkuesi** | Assembler | Translator converting assembly language to machine code; one-to-one correspondence |

### Programming & Algorithms (Programimi & Algoritmet)

| **Albanian Term** | **English Equivalent** | **Definition/Usage** |
|---|---|---|
| **Algoritm** | Algorithm | Well-defined set of ordered commands/steps following a specific path to solve a problem |
| **Algoritmika** | Algorithmics | Study and analysis of algorithms |
| **Kompleksiteti i algoritmeve** | Algorithm Complexity | Measure of efficiency; complexity in **time** (T(n)) and in **space** (memory usage) |
| **Kompleksiteti në kohë** | Time Complexity | Number of operations/steps required as function of input size |
| **Kompleksiteti në hapësirë** | Space Complexity | Amount of memory required as function of input size |
| **Rasti më i mirë** | Best Case | Minimum operations needed for algorithm to complete |
| **Rasti më i keq** | Worst Case | Maximum operations needed for algorithm to complete |
| **Rasti mesatar** | Average Case | Expected number of operations for typical input |
| **Përzgjedhja e algoritmeve** | Algorithm Selection | Choosing appropriate algorithm based on problem requirements and efficiency needs |
| **Algoritmet e llojit "Përça dhe Sundo"** | Divide & Conquer Algorithms | Approach based on recursion; consists of three steps: divide problem, conquer subproblems, combine solutions |
| **MergeSort** | MergeSort | Divide-and-conquer sorting algorithm; O(n log n) complexity |
| **QuickSort** | QuickSort | Divide-and-conquer sorting algorithm; efficient average-case performance |
| **Renditja** | Sorting | Arranging elements in specific order (ascending/descending) |
| **Kërkimi** | Searching | Finding specific element in data structure |
| **Ndryshorja / Variabla** | Variable | Named storage location holding value that can change during program execution |
| **Emra ndryshore** | Variable Names | Identifiers for variables; in Python: start with letter or underscore, contain alphanumeric/underscore, case-sensitive |
| **Tipi i të dhënave** | Data Type | Classification of data: integer, float, string, boolean, etc. |
| **Kontrollimi i rrjedhës së programeve** | Program Flow Control | Directing execution order of program statements |
| **Udhëzimi if** | if statement | Conditional execution; executes code block if condition is true |
| **if...else** | if...else statement | Two-branch conditional; executes one block if true, another if false |
| **if...elif...else** | if...elif...else statement | Multi-branch conditional; tests multiple conditions in sequence |
| **Cikli for** | for loop | Iterates fixed number of times; uses loop variable |
| **Cikli while** | while loop | Iterates while condition is true; condition checked before each iteration |
| **Kufizim i ciklus** | Loop Boundary | Start and end values defining iterations |
| **Nesting / Afrimi** | Nesting | Placing one control structure inside another (nested loops, nested if statements) |
| **Përpunimi** | Processing | Manipulating data through calculations and logical operations |
| **Python** | Python | High-level, interpreted programming language; widely used for education and data science; featured in this textbook |
| **Funksion** | Function | Reusable block of code performing specific task; has name, parameters, return value |
| **Parametrat e funksionit** | Function Parameters | Input values passed to function; formal parameters defined in function definition |
| **Vlera kthimi / Return value** | Return Value | Output produced by function |
| **Thirrja e funksionit** | Function Call | Executing function with specific arguments |
| **Deklarata e funksionit** | Function Definition | Specifying function name, parameters, and code body |
| **Funsionet e gatshme** | Built-in Functions / Library Functions | Pre-written functions available in language; examples: len(), range(), append(), remove() |
| **append()** | append() | List function adding element to end |
| **remove()** | remove() | List function removing first occurrence of element |
| **pop()** | pop() | List function removing and returning last element |
| **len()** | len() | Function returning length of sequence |
| **range()** | range() | Function generating sequence of numbers; used in for loops |
| **String / Varg karakteresh** | String | Data type representing sequence of text characters; immutable in Python |
| **Indeksimi** | Indexing | Accessing element by position; in Python, index starts at 0 |
| **Indeksimi negativ** | Negative Indexing | Accessing from end; -1 is last element |
| **Struktura e të dhënave** | Data Structure | Organized way of storing and managing data; examples: lists, stacks, queues |
| **Tabela / Lista** | List / Array | Ordered collection of elements; mutable in Python |
| **Pila** | Stack | Last-In-First-Out (LIFO) data structure; push/pop operations |
| **Radha** | Queue | First-In-First-Out (FIFO) data structure; enqueue/dequeue operations |
| **push()** | push() | Stack operation adding element to top |
| **pop()** | pop() | Stack operation removing element from top |
| **enqueue()** | enqueue() | Queue operation adding element to rear |
| **dequeue()** | dequeue() | Queue operation removing element from front |
| **LIFO** | Last-In-First-Out | Principle of stack operations |
| **FIFO** | First-In-First-Out | Principle of queue operations; also LILO (Last-In-Last-Out) |
| **Testimi i kodeve** | Testing Code | Verifying program correctness with various input data |
| **Korrigjimi / Debugging** | Debugging | Finding and fixing errors (bugs) in code |
| **Gabim sintaksor** | Syntax Error | Violation of programming language rules; prevents compilation |
| **Gabim logjik** | Logic Error | Code runs but produces incorrect results |
| **Ripërdorimi i kodit** | Code Reuse | Using existing code modules/functions in new projects; increases efficiency |
| **Libraritë** | Libraries | Collections of pre-written, tested functions available for reuse |
| **Algjebra buleane** | Boolean Algebra | Mathematical system of logic using true/false values |
| **Veprimet buleane** | Boolean Operations | AND, OR, NOT operations on boolean values |
| **AND** | AND | Boolean operation; true only if both conditions true |
| **OR** | OR | Boolean operation; true if at least one condition true |
| **NOT** | NOT | Boolean operation; negation; reverses truth value |
| **Tabela e vërtetësisë** | Truth Table | Table showing all possible boolean operation results |
| **Hartat Karnough** | Karnaugh Maps | Graphical tool for simplifying boolean expressions |
| **Sistemi inteligjent** | Intelligent System | Computer system mimicking human intelligence; includes AI, machine learning |
| **Inteligjenca artificiale (AI)** | Artificial Intelligence | Technology enabling computers to perform tasks requiring human-like intelligence |
| **Rrjetet nervore** | Neural Networks | AI systems inspired by biological neural networks |
| **Mësim i makinave** | Machine Learning | AI branch enabling systems to learn from data without explicit programming |

### Databases (Bazat e Të Dhënave)

| **Albanian Term** | **English Equivalent** | **Definition/Usage** |
|---|---|---|
| **Bazë të dhënash** | Database | Organized collection of structured data stored and accessed electronically; enables efficient storage and retrieval |
| **Bazat e të dhënave relacionale** | Relational Database | Database structured as related tables with rows and columns; follows relational model |
| **Modeli relacionale** | Relational Model | Database design model based on relations (tables); introduced by E.F. Codd |
| **SABShR (Sistemi i Administrimit të Bazave të Dhënash Relacionale)** | RDBMS (Relational Database Management System) | Software managing relational databases; examples: MySQL, PostgreSQL, MS SQL Server |
| **SQL** | SQL (Structured Query Language) | Standard language for accessing/manipulating relational databases; used in SABShR |
| **Tabela** | Table / Relation | Primary unit of relational database; two-dimensional structure of rows and columns |
| **Fusha / Kolona** | Field / Column / Attribute | Named column in table representing data characteristic; has data type and constraints |
| **Rreshti / Regjistrim** | Row / Record / Tuple | Complete set of values for all fields in table; represents single entity |
| **Çelës kryesor (Primary Key)** | Primary Key | Field(s) uniquely identifying each record in table; no duplicates/null values allowed |
| **Çelës i huaj (Foreign Key)** | Foreign Key | Field(s) referencing primary key in another table; maintains referential integrity; enables relationships |
| **Indeks** | Index | Data structure accelerating data retrieval; organized subset of table fields |
| **Radhë përpunimi** | Query | Request for specific data from database using SQL |
| **Marrëdhënie / Relacion** | Relationship | Connection between tables via primary and foreign keys |
| **Marrëdhënie një-në-një** | One-to-One Relationship | Each record in Table A corresponds to exactly one in Table B |
| **Marrëdhënie një-në-shumë** | One-to-Many Relationship | Each record in Table A corresponds to multiple in Table B |
| **Marrëdhënie shumë-në-shumë** | Many-to-Many Relationship | Records in Table A correspond to multiple in Table B and vice versa |
| **Integritet referencial** | Referential Integrity | Rule ensuring foreign key values reference valid primary keys; prevents invalid relationships |
| **Normalizimi** | Normalization | Process organizing database structure to reduce redundancy and improve integrity |
| **Forma e parë normale (1NF)** | First Normal Form (1NF) | Atomic attributes; each field contains single value, no repeating groups |
| **Forma e dytë normale (2NF)** | Second Normal Form (2NF) | In 1NF and non-key attributes fully dependent on primary key |
| **Forma e tretë normale (3NF)** | Third Normal Form (3NF) | In 2NF and non-key attributes not transitively dependent on primary key |
| **Redundanca e të dhënave** | Data Redundancy | Unnecessary duplication of data; eliminated through normalization |
| **Integritet i bazave të të dhënave** | Database Integrity | Accuracy and consistency of data; maintained through constraints and rules |
| **Kufizime** | Constraints | Rules limiting data values in fields (NOT NULL, UNIQUE, CHECK, DEFAULT) |
| **Fazat e zhvillimit të bazës së të dhënave** | Database Development Phases | Planning → Analysis → Design → Implementation → Testing → Deployment |
| **Diagramat E-R** | Entity-Relationship Diagrams (ERD) | Visual representation of database entities and relationships |
| **Bazat e të dhënave të shpërndara** | Distributed Databases | Database distributed across multiple computers/locations; improves availability and performance |
| **Replikimi** | Replication | Copying database to multiple locations for redundancy and availability |
| **Big Data** | Big Data | Very large datasets requiring advanced analysis techniques; characterized by volume, velocity, variety |
| **Analiza e të dhënave** | Data Analysis | Process extracting insights from data; using statistical and computational methods |

### Networking & Internet (Rrjetet & Interneti)

| **Albanian Term** | **English Equivalent** | **Definition/Usage** |
|---|---|---|
| **Rrjete kompjuterike** | Computer Network | Connected computers and devices sharing resources and information |
| **Interneti** | Internet | Global system of interconnected networks using TCP/IP protocol |
| **Transmetim i të dhënave** | Data Transmission | Sending data between computers over network |
| **Paketa** | Packet | Unit of data transmitted; includes header with addressing information and payload |
| **Komutim i paketave** | Packet Switching | Network technology dividing messages into packets transmitted separately and reassembled at destination |
| **Kontroll i gabimeve** | Error Control | Mechanisms detecting and correcting errors during transmission |
| **Zbulimi i gabimit** | Error Detection | Identifying transmission errors without correction |
| **Korrigjimi i gabimit** | Error Correction | Identifying and fixing transmission errors automatically |
| **Kodi CRC** | CRC Code (Cyclic Redundancy Check) | Mathematical error detection code appended to data |
| **ARQ (Automatic Repeat reQuest)** | ARQ (Automatic Repeat reQuest) | Error control using acknowledgment messages and retransmission; asks for repetition of failed transmission |
| **FEC (Forward Error Correction)** | FEC (Forward Error Correction) | Error control adding redundant bits enabling receiver to correct errors without retransmission |
| **Kuantizim** | Quantization | Converting continuous signal to discrete levels; approximates information with finite codewords |
| **Kompresim / Ngjeshje** | Data Compression | Reducing file size through mathematical algorithms |
| **Kompresim me humbje** | Lossy Compression | Compression discarding some data; smaller size but data loss; used for images, video, audio |
| **Kompresim pa humbje** | Lossless Compression | Compression preserving all original data; larger size than lossy; used for text, documents |
| **TCP/IP** | TCP/IP (Transmission Control Protocol/Internet Protocol) | Suite of protocols for internet communication; four-layer model: Application, Transport, Internet, Link |
| **Shtresa aplikimi** | Application Layer | Highest layer (TCP/IP); includes HTTP, FTP, SMTP, DNS protocols |
| **Shtresa transporti** | Transport Layer | Layer handling end-to-end communication; includes TCP, UDP protocols |
| **Shtresa interneti** | Internet Layer | Layer handling routing and logical addressing; includes IP, ICMP, IGMP protocols |
| **Shtresa lidhje** | Link Layer | Lowest layer; handles physical transmission; includes hardware addressing, MAC protocols |
| **Protocol** | Protocol | Set of rules governing communication between devices |
| **HTTP** | HTTP (HyperText Transfer Protocol) | Protocol for transferring web pages |
| **HTTPS** | HTTPS (HTTP Secure) | Encrypted version of HTTP using SSL/TLS |
| **FTP** | FTP (File Transfer Protocol) | Protocol for transferring files between computers |
| **SMTP** | SMTP (Simple Mail Transfer Protocol) | Protocol for sending email |
| **POP3** | POP3 (Post Office Protocol 3) | Protocol for retrieving email from server |
| **DNS** | DNS (Domain Name System) | System translating domain names to IP addresses |
| **TCP** | TCP (Transmission Control Protocol) | Connection-oriented, reliable transport protocol; ensures all data arrives in order |
| **UDP** | UDP (User Datagram Protocol) | Connectionless transport protocol; faster but less reliable than TCP |
| **Adrese IP** | IP Address | Unique numerical identifier for device on network; 32-bit (IPv4) or 128-bit (IPv6) |
| **IPv4** | IPv4 (IP version 4) | Standard IP addressing; 32-bit addresses; written in dotted decimal notation (e.g., 192.168.1.1) |
| **IPv6** | IPv6 (IP version 6) | Newer IP standard; 128-bit addresses; supports more devices |
| **Adresa IP statike** | Static IP Address | Permanent, fixed IP address assigned to device |
| **Adresa IP dinamike** | Dynamic IP Address | Temporary IP address assigned from pool; changes when device reconnects |
| **Klasat e rrjetave** | Network Classes | IPv4 address classes (A, B, C, D, E) defining address ranges |
| **Mask-a rrjeti** | Network Mask / Subnet Mask | Defines portion of IP address identifying network |
| **Gateway** | Gateway | Device connecting different networks; routes traffic between them |
| **Router** | Router | Device directing data packets between networks based on IP addresses |
| **Switch** | Switch | Device connecting devices within network; operates at data link layer |
| **Hub** | Hub | Device connecting multiple network segments; broadcasts all traffic to all ports |
| **Rrjete lokale (LAN)** | Local Area Network (LAN) | Network within limited geographic area; high speed, owned by organization |
| **Rrjete të gjerë (WAN)** | Wide Area Network (WAN) | Network spanning large geographic area; interconnects LANs; lower speed |
| **Rrjete metropolitan (MAN)** | Metropolitan Area Network (MAN) | Network covering city or large campus; larger than LAN, smaller than WAN |
| **Interneti i Gjërave (IoT)** | Internet of Things (IoT) | Concept of everyday objects connected to internet with sensors; enables communication and data sharing |
| **Sensor** | Sensor | Device detecting physical phenomena and converting to electronic signals |
| **Aktuetor** | Actuator | Device controlled by computer to perform physical action |
| **Aplikacion IoT** | IoT Application | Use of connected devices for monitoring, automation, control |
| **Siguria e IoT** | IoT Security | Protecting IoT systems from cyber attacks; challenges due to limited processing power |

   - Uses imperative forms for instructions: "Shkruaj..." (Write...), "Testo..." (Test...)

4. **Bilingual Glossaries**
   - Each concept box lists terms in **both Albanian and English**
   - Key concepts are often introduced in English first, then explained in Albanian
   - Example: "Arkitektura Von Neumann (Von Neumann Architecture)"

5. **Compound Words with Prepositions**
   - Extensive use of compound descriptions using "i/e" (of):
     - "Sistemi operative" (Operating System)
     - "Memoria kryesore" (Primary Memory)
     - "Rrjeta lokale" (Local Network)
     - "Baza të dhënash" (Database, literally "base of data")
     - "Përpunim paralel" (Parallel Processing)

6. **Algorithm & Programming in Plain Albanian**
   - Algorithmic concepts translated more fully:
     - "Përça dhe Sundo" (Divide and Conquer)
     - "Rrjedhje programi" (Program Flow)
     - "Kontroll i kushtëzuar" (Conditional Control)
     - "Përsëritje" (Iteration/Repetition)
     - "Ndryshorje" (Variable)
     - "Funksion" (Function)

7. **Database Terminology: Fully Localized**
   - Strong preference for Albanian terms:
     - "Baza të dhënash" (Database)
     - "Tabela" (Table)

## Common Translation Traps & Important Distinctions

| **Area** | **Trap/Issue** | **Correct Albanian Usage** |
|---|---|---|
| **Variables vs. Data** | Don't confuse "ndryshorje" (variable) with "të dhëna" (data). Variables are containers; data is content. | "Variabla përmbajn të dhëna" (Variable holds data) |
| **Algorithm vs. Program** | "Algoritm" is abstract steps; "program" is executed code. Both are distinct concepts. | Algorithm is abstract → Code is program → Execution is execution |
| **Memory Types** | Conflating primary (volatile) with secondary (persistent). Key distinction for computer architecture. | "Memoria kryesore" (RAM, volatile) vs. "Memoria sekondare" (persistent storage) |
| **Encryption Terms** | "Kodifikimi" can mean both encoding and encryption; context matters. | Use "enkriptim" specifically for security encryption |
| **Open Source** | "Burim i hapur" (open source) is sometimes confused with "falas" (free); open source can have licensing restrictions. | "Burim i hapur" ≠ necessarily "falas" |
| **Interface** | "Ndërfaqja" can mean UI or architectural boundary. Clarify context. | UI context: "ndërfaqe përdoruesi"; technical context: "ndërfaqe" between layers |
| **Query vs. Statement** | SQL "query" specifically refers to SELECT; "statement" includes INSERT/UPDATE/DELETE. In Albanian, often just "radhë përpunimi" (query) for all. | Distinction important in database contexts |
| **Loop Iteration** | "Cikli" (loop) vs. "përsëritje" (iteration); in Python "for" loops, the variable iterates over values. | "Variabla in range()" iterates; it doesn't "increment" necessarily |
| **Class vs. Type** | Not often distinguished in intro texts; "klasa" (class) and "tipi i të dhënave" (data type) are related but different. | In OOP: "klasa" is blueprint; in typing: "tipi" is classification |
| **Firewall Rules** | Often described simply as filtering; context of stateful vs. stateless firewalls rarely discussed. | Default: "firewall e monitoron trafikun" (monitors traffic) and blocks per rules |
| **IP Addressing** | "Adresa IP" can refer to both IPv4 and IPv6; students often don't realize these are different standards. | Specify "IPv4" (32-bit) vs. "IPv6" (128-bit) when precision matters |
| **Deployment vs. Testing** | Sometimes conflated; "shpërndarje" (deployment) ≠ "testim" (testing). | Testing phase → Deployment phase (distinct) |

## Useful Sentence Patterns & Explanatory Structures

### Definition Pattern
**"X quhet Y, që..."** (X is called Y, which...)
- *CPU quhet Njësia Qendrore e Përpunimit, që është përgjegjëse për ekzekutimin e udhëzimeve.*
- (CPU is called the Central Processing Unit, which is responsible for executing instructions.)

### Functional Pattern
**"Qëllimi i X-it është të Y"** (The purpose/goal of X is to Y)
- *Qëllimi i CPU-së është të përpunojë të dhënat.*
- (The purpose of the CPU is to process data.)

### Characteristic Pattern
**"X karakterizohet nga Y"** (X is characterized by Y)
- *Memoria primare karakterizohet nga shpejtësia e lartë dhe kapaciteti i limituar.*
- (Primary memory is characterized by high speed and limited capacity.)

### Process Step Pattern
**"Hapi i parë/dytë/tretë të X është Y"** (The first/second/third step of X is Y)
- *Hapi i parë i ciklit fetch-execute është marrja e udhëzimit.*
- (The first step of the fetch-execute cycle is instruction fetch.)

### Comparison Pattern
**"Ndryshe nga X, Y..."** (Unlike X, Y...)
- *Ndryshe nga memoria kryesore, memoria sekondare nuk humb të dhënat kur kompjuteri fiket.*
- (Unlike primary memory, secondary memory doesn't lose data when the computer shuts down.)

### Advantage/Disadvantage Pattern
**"Përparesia e X-it është Y" / "Mangësia e X-it është Y"**
- *Përparesia e modelit Waterfall është planifikimi i detajuar; mangësia është infleksibiliteit ndaj ndryshimeve.*
- (The advantage of the Waterfall model is detailed planning; the disadvantage is inflexibility to changes.)

### Conditional Pattern
**"Nëse X, atëherë Y"** (If X, then Y)
- *Nëse të dhënat nuk janë në formën e tretë normale, baza e dhënash nuk garanton integritetin e të dhënave.*
- (If the data is not in third normal form, the database doesn't guarantee data integrity.)

### Question-Answer Pattern
**Used extensively in practice sections:**
- *Çfarë është X? → X quhet...*
- *Si funksionon X? → X funksionon kur...*
- *Cilat janë karakteristikat e X-it? → Karakteristikat e X-it janë...*

### Explanation with Examples
**"p.sh." (për shembull = for example)**
- *Ekzistojnë lloje të ndryshme të memories, si p.sh. memoria primare dhe ajo sekondare.*
- (There are different types of memory, such as primary and secondary memory.)

### Causative Pattern
**"X ndikon në Y" / "X shkakton Y"** (X affects Y / X causes Y)
- *Shpejtësia e CPU-së ndikon shumë në performancën e kompjuterit.*
- (CPU speed greatly affects computer performance.)

### Negation Pattern
**"X nuk është Y" / "X nuk funksionon me Y"** (X is not Y / X doesn't work with Y)
- *Memoria RAM nuk është e qëndrueshme; humb të dhënat kur kompjuteri fiket.*
- (RAM is not persistent; it loses data when the computer shuts down.)

## Sentence Structures in Technology Albanian

### Pattern 1: Definition with "është" (is)
**Structure**: Noun + is + descriptive noun/phrase
**Examples from textbook**:
- "SRAM-i është një memorie e shpejtë." — SRAM is fast memory.
- "Memoria e përhershme ROM mban të dhënat." — Permanent ROM memory holds the data.
- "CPU-ja është pjesa më e rëndësishme e kompjuterit." — The CPU is the most important part of the computer.

### Pattern 2: Comparison using "më i/e" (more) comparative structures
**Structure**: X + "më i" + adjective + "se" (than) + Y
**Examples from textbook**:
- "SRAM-i është shumë më i shpejtë se DRAM-i." — SRAM is much faster than DRAM.
- "SRAM-i është më i ndërlikuar, prandaj më i shtrenjtë." — SRAM is more complex, therefore more expensive.

### Pattern 3: Causal relation with "prandaj" (therefore)
**Structure**: [Characteristic], prandaj [Consequence]
**Examples from textbook**:
- "SRAM-i është më i ndërlikuar, prandaj është më i shtrenjtë." — SRAM is more complex, therefore it is more expensive.
- "Ai nuk mund të ndryshojë, prandaj quhet memorie e përhershme." — It cannot change, therefore it is called permanent memory.

### Pattern 4: Purpose/Function with "për + infinitive" (for/to)
**Structure**: [Item] + [verb] + për të [action]
**Examples from textbook**:
- "PROM përdoret në pajisjet elektronike për të ruajtur të dhëna." — PROM is used in electronic devices to store data.
- "EPROM mund të fshihet për të rinovojtur të dhënat." — EPROM can be erased to renew the data.
- "Kompjuteri përdor CPU-në për të përpunuar të dhënat." — The computer uses the CPU to process data.

### Pattern 5: Passive Voice for Technical Descriptions
**Structure**: [Object] + passive verb + [agent/manner]
**Examples from textbook**:
- "PROM-i programohet një herë pas prodhimit." — PROM is programmed once after manufacturing.
- "EPROM mund të fshihet duke e ekspozuar ndaj dritës ultravjollcë." — EPROM can be erased by exposing it to ultraviolet light.
- "Të dhënat shkruhen në një ROM gjatë prodhimit." — Data is written in a ROM during manufacturing.

### Pattern 6: Classification with "mund të jetë" (can be)
**Structure**: [General category] + "mund të jetë" + [Type 1] ose/apo [Type 2]
**Examples from textbook**:
- "Memoria ROM mund të jetë PROM ose EPROM." — ROM memory can be PROM or EPROM.
- "Memoria mund të jetë kryesore ose sekondare." — Memory can be primary or secondary.

### Pattern 7: Function description with "shërbim" (purpose/function)
**Structure**: [Item] + [verb] + për [purpose]
**Examples from textbook**:
- "Ai shfrytëzohet për memoriet e shpejta." — It is used for fast memories.
- "BIOS-i shërbim për të kontrolluar pjesët hardware." — BIOS serves to control hardware components.

### Pattern 8: Negation with "nuk...jo" or "nuk...por" structures
**Structure**: Nuk [negative] + [positive alternative]
**Examples from textbook**:
- "Ata nuk e humbasin ngarkimin gjatë përdorimit." — They don't lose the charge during use.
- "Zakonisht nuk ndryshojnë, por ruhen përgjithmonë." — They usually don't change, but are stored permanently.

### Pattern 9: Participial phrases in technical description
**Structure**: [Noun] + [participle] + [additional info]
**Examples from textbook**:
- "Memoria e përhershme ROM, mban të dhënat dhe udhëzimet." — Permanent ROM memory, [which] maintains the data and instructions.
- "Procesorët me shumë bërthama, punojnë në mënyrë paralele." — Multi-core processors, [which] work in parallel.

### Pattern 10: Sequential steps with "Pastaj/Më pas" (Then/Next)
**Structure**: [Step 1]... Pastaj [Step 2]
**Examples from textbook**:
- "Të dhënat programohen në pajisje. Pastaj nuk mund të ndryshohen." — Data is programmed into devices. Then it cannot be changed.

### Pattern 11: Specification with "lloj" (type/kind)
**Structure**: [General] + "lloj" + [specific]
**Examples from textbook**:
- "EPROM është një lloj i memorieve ROM." — EPROM is a type of ROM memory.
- "Kompjuteri ka disa lloje memorje." — The computer has several types of memory.

### Pattern 12: Naming with "quhet" (is called)
**Structure**: [Concept] + "quhet" + [Name]
**Examples from textbook**:
- "Ky lloj i ndërfaqes quhet ndërfaqja grafike e përdoruesit (GUI)." — This type of interface is called the graphical user interface (GUI).
- "Ky proces quhet boot process." — This process is called the boot process.

### Pattern 13: Manner description with "përmes" (through/by means of)
**Structure**: [Action] + përmes [Means]
**Examples from textbook**:

## Grammar Context in Technology Texts

### 1. Gender Assignment to Technical Terminology
Technology texts assign systematic gender to computing terms:
- **Masculine**: CPU-ja (from procesori), ROM-i, RAM-i, bus-i, bit-i, byte-i
- **Feminine**: memoria, pajisja, ndërfaqja, drita, tastiera, miu (mouse)
- Complex terms follow the gender of the head noun: "memoria kryesore" (fem), "sistema operativ" (masc)
- Example: "SRAM-i është shumë më i shpejtë se DRAM-i" — Both masculine

### 2. Compound Technical Terms and Fixed Collocations
Technology Albanian uses specific compound structures:
- **Adjective-noun compounds**: "memorie volatile", "dhënë binare" (binary data), "kod burimor" (source code), "burim i hapur" (open source)
- **Noun-noun compounds**: "tranzistor elektronik", "ndërfaqja grafike", "memoria sekondare"
- These compounds often require genitive constructions: "memoria e shpejtë", "dhënë e përhershme"

### 3. Definite vs. Indefinite Articles in Technical Description
Technology texts show clear article patterns:
- **General/universal concepts**: "Memoria ROM mban të dhënat" (Memory ROM holds data) - definite
- **When introducing new concepts**: "EPROM është një lloj i memories" (EPROM is a type of memory) - indefinite
- **With loan words**: "një ROM", "një CPU", "një GUI" — indefinite used even with established terms
- When discussing specific instances: "Ky ROM mban udhëzimet e sistemit" (This ROM holds system instructions) - definite

### 4. Extensive Use of Passive Voice
Technical writing heavily employs passive voice for objective description:
- "PROM-i programohet..." (PROM is programmed...)
- "Të dhënat shkruhen..." (Data is written...)
- "EPROM mund të fshihet..." (EPROM can be erased...)
- This creates objective, non-agent-centered technical discourse
- Example: "Memoria është programuar para se të dërgohet" — Memory is programmed before being sent (agent unknown/irrelevant)

### 5. Genitive Chains for Component Relationships
Technology texts use genitive constructions to show hierarchical relationships:
- "pjesa më e rëndësishme e kompjuterit" (the most important part of the computer)
- "bërthama e procesorit" (core of the processor)
- "memoria e kompjuterit" (computer's memory)
- "udhëzimet e sistemit" (system's instructions)
- "kapaciteti i memorjes" (memory capacity)
- Example: "BIOS-i mban udhëzimet e kontrollit të hardware-it" — BIOS holds hardware control instructions

### 6. Integration of English/Technical Terms with Albanian Grammar
English computing terms are systematized in Albanian:
- Acronyms take Albanian articles and gender: "CPU-ja" (fem), "ROM-i" (masc), "RAM-i" (masc)
- Mixed phrases: "memoria RAM", "procesor Intel", "sistema Windows"
- Loan words are sometimes Albanized: "transistor" → "tranzistor", "program" → "program"
- Example: "CPU-ja përpunon të dhënat" — The CPU processes data (English term takes Albanian grammar)

### 7. Adjective Agreement in Technical Description
Technical adjectives strictly agree with noun gender and number:
- **Singular**: "memorie volatile" (volatile memory - fem), "procesor i shpejtë" (fast processor - masc)
- **Plural**: "memoria e shpejta" (fast memories - fem pl), "procesorët e rinj" (new processors - masc pl)
- Comparative adjectives: "më i shpejtë se" (faster than), "më i ndërlikuar" (more complex)
- Example: "Memoriet elektronike janë shumë më të shpejta" — Electronic memories are much faster

### 8. Relative Clause Structure in Technical Explanation
Technology texts embed relative clauses for specification:
- "Memoria ROM, e cila mban të dhënat..." — ROM memory, which holds data...
- "Procesorët, të cilët punojnë në mënyrë paralele..." — Processors, which work in parallel...
- Relative marker agrees with antecedent gender: e (fem), i (masc), të (pl)
- Example: "BIOS-i, i cili është program specializ, kontrollon hardware-in" — BIOS, which is a specialized program, controls hardware

### 9. Nominalization of Verbs in Technical Terminology
Technology writing creates noun forms for processes and functions:
- "përpunim" (processing - from përpunon)
- "ruajtje" (storage - from ruan)
- "përshpejtim" (acceleration - from përshpejton)
- "ndërhyrje" (interference - from ndërhyr)
- "programim" (programming - from programon)
- Example: "Përpunimi i të dhënave ndodh në CPU-ë" — Data processing occurs in the CPU

### 10. Prepositional Phrases for Technical Relationships
Technology texts use specific prepositions for different relationships:
- **"në"** (in/at): "në memorjen ROM" (in ROM memory), "në sistem" (in the system)
- **"me"** (with): "memoria me kapacitet të lartë" (memory with high capacity)
- **"përmes"** (through): "përmes ndërfaqes" (through the interface)
- **"nga"** (by/from): "e programueshme nga përdoruesi" (programmable by the user)
- **"për"** (for): "memoria për ruajtjen e të dhënave" (memory for data storage)
- Example: "Ndërhyrja nga jashtë mund të dëmtojë sistemin" — Interference from outside can damage the system
